Tags HTML para texto

No Html, tags podem conter outras tags, Mas pra isso é preciso entender a diferença entre "block" e "InLine".

Block e InLine

No HTML, os elementos podem ser classificados como block ou inline, e cada um tem um comportamento visual distinto:

Elementos Block

- Comportamento: Ocupam toda a largura disponível e sempre começam em uma nova linha.

- Exemplos: < div >, < h1 > - < h6 >, < p >, < blockquote >, < ul >, < ol >, < form >.

- Uso: São usados para criar seções maiores e estruturais na página.

Elementos InLine

- Comportamento: Ocupam apenas o espaço necessário e não iniciam uma nova linha.

- Exemplos: span, a, img, strong, em.

- Uso: São usados para estilizar partes menores do conteúdo dentro de elementos block.

Diferença Principal

- Block: Elementos block são empilhados verticalmente, um abaixo do outro.

- Inline: Elementos inline são dispostos horizontalmente, lado a lado, dentro de um elemento block.

Tag < b > (bolt) e < strong >

- < b > (bolt): destaca uma palavra ou texto visualmente em negrito. Não tem significado extra ao texto(semantica), é puramente estético.
Ex:
eu estou em negrito.

- < strong >: deixa uma palavra ou texto mais forte. Adiciona significado ao texto, indicando que ele é importante, sério ou urgente.
Ex:
Eu tenho mais importancia.

Diferença principal

- Visual: Ambos são renderizados em negrito por padrão.

- Semântica: < b > é para destaque visual sem importância adicional, enquanto < strong > indica que o texto é de grande importância.


< i > (italic) e < em > (emphasis)

- < i > (italic): Utilizada para destacar texto que deve ser diferenciado do restante, como termos técnicos, expressões em outros idiomas ou pensamentos de personagens fictícios. Não adiciona significado extra (semantica), é puramente estético.
Ex:
Eu estou em italico.

- < em > (emphasis): Utilizada para indicar que o texto tem ênfase. Adiciona significado ao texto, indicando que ele é importante ou deve ser lido com ênfase.
Ex:
Eu tenho ênfase.

Diferença principal

- Visual: Ambos são renderizados em itálico por padrão.

- Semântica: < i > é para destaque visual sem importância adicional, enquanto < em > indica que o texto é enfatizado e tem um significado importante.
